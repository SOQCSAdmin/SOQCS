SOQCS  RV1.0 (Beta)
Stochastic Optical Quantum Circuit Simulator
Compilation and installation

Requirements

  • Linux/Unix operating system.
  • C++ compiler.
  • GNU Make.
  • Eigen 3 library. (Follow the instruction installation below)

How to build it?

Step 1: Download the library.

Download the library from the github folder.

Step 2: Decompress

Decompress the zip file with your favorite decompressor or type in the command line:

unzip "soqcs zip filename".zip

and move the resulting library folder to the desired location.

Step 3: Install Eigen 3.

SOQCS needs Eigen 3 external library to be built. Eigen 3 can be automatically downloaded and installed for SOQCS using the script in SOQCS root folder:

/"SOQCS root folder"/config.sh

Note for advanced users: The script is configured to download V3.4 of Eigen. If V3.4 becomes unavailable the script can be reconfigured for other versions changing the number in the configuration variable VER within the script.

Step 3 (alternate): Install Eigen 3 manually.

If the automated installation of Eigen does not work Eigen 3 library can be installed manually following the next steps.

  • Download Eigen 3 library manually from its homepage.
  • Decompress the library.
  • Copy the subfolder Eigen to its corresponding location in the SOQCS building source tree.
    cp -R /"Eigen library folder"/Eigen/ * /"SOQCS root folder"/src/Eigen/

Step 4: Build the library and all the examples.

Inside the SOQCS library main folder type make. This will build the library and all the examples. The examples may be found in the subfolder /SOQCS root folder/examples.

Step 4 (alternate): Build only the library.

Inside the library main folder go to /SOQCS root folder/src folder and type make. This will build only the library without the examples.

Troubleshooting

If some of the previous steps fail it is recommended to check the configuration file conf.inc in SOQCS root directory. If the command of the compiler or the linker in your computer has a different name or some compilation flags are troublesome in your system they can be reconfigured in this file.

How to use it?

Quick start in C++

To build a SOQCS simulation you can create or modify a file named main.cpp in the folder /SOQCS root folder/devel. This file has to contain:

#include "soqcs.h"
main{
"Your simulation instructions"
}

Type make within that folder to build the simulation and execute it with ./main.x. You will find a few guidelines about how to program in SOQCS below. Additionally your will find some examples in the examples folder.

Quick start in Python

To build a SOQCS simulation in Python you can create or modify a file named main.py in the folder /SOQCS root folder/devel. This file has to contain:

import pysoqcs
"Your simulation instructions"

Use your preferred editor or command line client to execute it. Alternatively the library can also be used in jupyter notebooks.
The python port is still under development and it does not have yet all the same functionality than the parent C++ library. We recommend to look at the python examples and follow the same guidelines than the C++ programs.

Importing SOQCS to a project

SOQCS has two build targets:

  • libsoqcs.a : Is the C++ library
  • libsoqcs.so : Is the C++ side of the connection of the SOQCS port into Python


Importing SOQCS C++ library.
In your programs include the line

#include "soqcs.h"

and compile your program with

g++ your_program.cpp -I /"SOQCS root folder"/src -L /"SOQCS root folder"/src -lsoqcs -lpthread -O2 -std=c++17 -fopenmp -o executable_name.x

Alternatively you may copy libsoqcs.a and the .h files to another folder. In this case the program has to be compiled as

g++ your_program.cpp -I /"library folder path" -L /"library folder path" -lsoqcs -lpthread -O2 -std=c++17 -fopenmp -o executable_name.x


Importing SOQCS port in Python .
Copy pysoqcs.py and libsoqcs.so to your working folder and start your programs with,

import pysoqcs

these files can be found in the /SOQCS root folder/py_src folder
